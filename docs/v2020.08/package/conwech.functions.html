conwech.functions

Home to all of the fun stuff, which consists of two main function pairs, nameperiod() / readperiod() and number2text() / text2number(). As their names hopefully suggest, for both pairs, each function is designed to be the other’s inverse (to the extent possible). The intended promise is that calling one with the output of the other will return the input of the other, or at least something equivalent, which could be used the same way in the other direction.

nameperiod(zillion: int) → str[source]

Names the period for the given zillion value.

The function returns the Conway-Wechsler name for a number with the given zillion value. The zillion property of a number in the short-scale system is equal to one less than the number of periods in the number, where a period is a set of one to three consecutive digits often separated by commas.

More formally, the zillion value zn for any real number n can be obtained by the function Z(n):

Z(n) = ⌊log1000(n)⌋ - 1

Parameters:

zillion (int) – The zillion value of the period name.

Returns:

The name for the period with the given zillion value.

Return type:

str

Raises:

TypeError – If zillion is not an int.

Examples

>>> from conwech.functions import nameperiod
>>> nameperiod(-1)
''
>>> nameperiod(0)
'thousand'
>>> nameperiod(4)
'quadrillion'
>>> nameperiod(789)
'novemoctogintaseptingentillion'
>>> nameperiod(123456789)
'tresviginticentillisesquinquagintaquadringentillinovemoctogintaseptingentillion'
readperiod(name: str) → int[source]

Convert name to its corresponding zillion value.

The inverse function of nameperiod(), this function parses the given period name by indexing a tuple of valid Conway-Wechsler period prefix components with each separate component of the period name.

If name is an empty string, the function returns -1 for the units period (first period).

Parameters:

name (str) – The period name of any number period.

Returns:

The zillion value for the period with the given name.

Return type:

int

Raises:

InvalidPeriodNameText – If any sub-component of name is not in ZILLION_PERIOD_PREFIXES.

Examples

>>> from conwech.functions import readperiod
>>> readperiod('')
-1
>>> readperiod('thousand')
0
>>> readperiod('quadrillion')
4
>>> readperiod('novemoctogintaseptingentillion')
789
>>> readperiod('tresviginticentillisesquinquagintaquadringentillinovemoctogintaseptingentillion')
123456789
number2text(number: int | float | str) → str[source]

Construct the English short-scale spelling of number.

The function returns the english short-scale spelling for number which can be any positive or negative number passed as an integer, float, or string (as long as that string fits a valid numeric pattern; see NUMERIC_STRING).

The function can handle values that exceed traditional limitations on numerical types. Pass number as a string for values requiring more precision or values greater outside the minimum and maximum int/float values.

Note

The function will also (eventually) be able to interpret a pseudo-sum-like string, the format sometimes returned by text2number(). Currently, this functionality is not yet in place, but should be implemented soon…ish… Bite me.

Parameters:

number (Union[int, float, str]) – The number to spell.

Returns:

The spelling of the given number.

Return type:

str

Raises:

InvalidNumericString – If number does not follow python’s conventional formatting for int and float types (see NUMERIC_STRING).

Examples

>>> from conwech.functions import number2text
>>> number2text(-123456)
'negative one hundred twenty-three thousand four hundred fifty-six'
>>> number2text(4.56e100)
'forty-five duotrigintillion six hundred untrigintillion'
>>> number2text('7.89e500')
'seven hundred eighty-nine quinquasexagintacentillion'
>>> number2text('-1.2e-9')
'negative twelve ten billionths'
text2number(text: str) → str[source]

Convert an English short-scale text to it’s numerical value.

The function attempts to parse text and return a numeric string containing the corresponding value. The number(s) in the string will be in scientific notation.

The function returns a pseudo-sum-like representation if any of the periods in the given text would result in adding an unreasonable amount of consecutive zeros.

The function will recognize fractions, but only those with a base-ten denominator, i.e. those which can be represented with decimal digits.

Parameters:

text (str) – The spelling of a number as a string.

Returns:

A numeric string representing the value of text.

Return type:

str

Raises:

  • InvalidNumeralString – If text does not match the required format (see NUMERAL_STRING).

  • InvalidPeriodValueText – If a period value in text is not in NATURAL_NUMBERS_LT_1000.

  • InvalidPeriodNameText – If any period name in text is not derived from a valid combination of Conway-Wechsler period prefixes (see readperiod()).

Examples

>>> from conwech.functions import text2number
>>> text2number('negative one hundred twenty-three one thousandths')
'-1.23456789e5'
>>> text2number('one millinillinillion and two one millinillinillionths')
'1e3000003 + 2e-3000003'
